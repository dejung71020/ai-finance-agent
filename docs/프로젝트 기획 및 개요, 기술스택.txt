"돈 관리를 자동화하고, 소비 심리를 교정하고, 자산을 성장시키는 — 나만의 금융 OS"
--------------------------------------------------------------------------------
1. 프로젝트 비전 (Why)
현재 금융 서비스의 문제점은 세 가지입니다.
파편화 — 은행, 증권, 카드, 코인, 포인트가 모두 따로 흩어져 있음
수동성 — 사용자가 직접 확인하고 판단해야 하는 구조
심리 무방비 — 소비 유혹에 대한 즉각적 개입 수단이 없음
AI 금융 비서는 이 세 가지를 동시에 해결합니다.
문제
해결 방향
파편화된 자산
전 금융권 API 통합 → 단일 대시보드
수동적 관리
n8n 자동화 + AI 에이전트로 자동 실행
심리적 취약
행동경제학 기반 실시간 소비 개입
--------------------------------------------------------------------------------
2. 핵심 목표 (What)
🎯 완전 자동화 — 사용자 개입 최소화
잔액 변화, 급여 입금, 소비 패턴에 따라 저축·투자·이체가 자동으로 실행됩니다. 사용자는 설정만 하면 됩니다.
🧠 심리적 가이드 — 행동경제학 적용
"이 커피 한 잔 = 10년 후 은퇴자금 3만원 손실" 같은 즉각적 맥락 피드백으로 소비 습관을 교정합니다.
📊 통합 가시성 — 모든 자산 한눈에
은행·카드·증권·코인·포인트까지 하나의 화면에서 실시간으로 파악합니다.

7. 데이터 흐름 핵심 이해
이 프로젝트를 이해할 때 가장 중요한 개념입니다.
📥 Pull 방식 vs Push(Webhook) 방식
Pull 방식 — Celery가 주기적으로 금융 API를 호출해서 데이터를 가져옴
장점: 안정적, 모든 기관 지원
단점: 실시간성 부족 (5분~1시간 지연)
Push(Webhook) 방식 — 거래 발생 시 기관이 직접 서버로 데이터를 보냄
장점: 실시간 (수초 이내)
단점: API 지원하는 기관만 가능
→ 두 방식을 병행해서 커버리지와 실시간성을 모두 확보합니다.
🔄 n8n이 하는 일
n8n은 조건 → 판단 → 액션 흐름을 코드 없이 연결합니다.
예시:
🧱 Layered Architecture 이해하기
왜 이렇게 나누나? → 각 레이어를 독립적으로 테스트하고 교체할 수 있어서.
예를 들어 DB를 PostgreSQL→MongoDB로 바꿔도 Repository만 수정하면 됩니다.
